# Bank Cards Management System

Backend для управления банковскими картами. REST API с JWT авторизацией, ролевым доступом (ADMIN/USER), переводами между картами и админ-панелью.

## Технологии

Java 17, Spring Boot 3.1.5, Spring Security (JWT), Spring Data JPA, PostgreSQL, Liquibase, Lombok, OpenAPI/Swagger, Docker, JUnit 5, Mockito, Maven

## Функционал

### Пользователи
- JWT аутентификация
- Роли: USER, ADMIN
- Автоматическое создание тестовых пользователей при первом запуске

### Карты
- Просмотр своих карт с пагинацией
- Фильтрация по статусу (ACTIVE, BLOCKED, EXPIRED)
- Поиск по номеру карты или имени владельца
- Просмотр баланса
- Запрос блокировки карты

### Переводы
- Переводы между своими картами
- Проверка достаточности средств
- История переводов
- Описание перевода

### Админ-панель
**Управление пользователями:**

  - Просмотр всех пользователей
  - Создание новых пользователей
  - Блокировка/разблокировка
  - Изменение роли (USER/ADMIN)
**Управление картами:**

  - Просмотр всех карт
  - Создание карт
  - Блокировка/активация
  - Удаление карт

### Безопасность
- Шифрование номеров карт
- Маскирование номеров (**** **** **** 1234)
- Ролевой доступ к эндпоинтам
- JWT токены для аутентификации

## Запуск

### Локальный запуск
`mvn spring-boot:run`
- API: [http://localhost:8080](http://localhost:8080)
- Swagger: [http://localhost:8080/swagger-ui.html](http://localhost:8080/swagger-ui.html)
### Запуск в Docker
`docker-compose up --build`

## API Examples

**Логин**
```
POST /api/auth/login
{
  "username": "admin",
  "password": "admin123"
}
```

**Основные операции**

- `GET    /api/cards/my`           — список карт (с пагинацией и фильтрами)
- `GET    /api/cards/my/{id}/balance` — баланс карты
- `POST   /api/cards/transfer`      — перевод между своими картами
- `POST   /api/cards/my/{id}/block`  — запрос блокировки

**Админ-панель**

- `GET    /api/admin/users`         — все пользователи
- `POST   /api/admin/users`          — создать пользователя
- `PATCH  /api/admin/users/{id}/status` — блокировка/разблокировка
- `GET    /api/admin/cards`          — все карты
- `POST   /api/admin/cards`          — создать карту
- `PUT    /api/admin/cards/{id}/block`   — заблокировать карту
- `DELETE /api/admin/cards/{id}`      — удалить карту

## Тестовые данные

При первом запуске автоматически создаются:

|Роль|Логин|Пароль|Описание|
|---|---|---|---|
|**ADMIN**|admin|admin123|Полный доступ|
|**USER**|user|user123|Обычный пользователь с 2 картами|

**Тестовые карты пользователя user:**
- Карта 1: **** **** **** 1234, баланс 1000.00
- Карта 2: **** **** **** 5678, баланс 500.00
